# anilistgen

## Summary

A CLI tool and background daemon that fetches anime for every season
of a given year from **AniList** and creates or updates corresponding
**MDBList** lists. Configured via a single YAML file, environment
variables, or both. Designed to run as a one-shot command, a scheduled
systemd timer, a long-lived daemon, or a container.

## Problem

Agregarr has no built-in "current season anime" source that captures
every show airing that season â€” only trending/popular subsets.
Community-maintained Trakt and MDBList lists for seasonal anime are
unreliable and often abandoned. This tool replaces manual list curation
by generating authoritative lists directly from AniList's seasonal data
and publishing them to MDBList, where Agregarr can consume them as
Custom List sources.

## Configuration

### Configuration sources (layered, later overrides earlier)

1. **Config file** (YAML) â€” searched in order:
   - Path specified via `-config` / `-c` CLI flag.
   - `./anilistgen.yaml` in the current working directory.
   - `$XDG_CONFIG_HOME/anilistgen/anilistgen.yaml` (defaults to
     `~/.config/anilistgen/anilistgen.yaml` if `XDG_CONFIG_HOME` is
     unset).
2. **CLI flags** â€” `-config`, `-dry-run`, `-output`, `-v`, `-h`.
3. **Environment variables** â€” every config field has an `ALG_`-prefixed
   env var. Values are comma-separated for lists, `true`/`1` for booleans,
   and Go duration strings for intervals.

### Config file structure

```yaml
# MDBList API key (required). Can also be set via ALG_MDBLIST_API_KEY env var.
mdblist_api_key: ""

# How often to run in daemon mode. Parsed as Go duration.
# Default: 24h. Set to 0 for one-shot mode.
interval: 24h

# Whether to run a sync immediately on daemon startup. Default: true.
run_on_start: true

# Path for last-run timestamp file. Default: /tmp/anilistgen.lastrun
state_file: /tmp/anilistgen.lastrun

# AniList query settings
anilist:
  # Single year to fetch. 0 = current year. Ignored if years is set.
  year: 0

  # Multiple years â€” overrides year above.
  # years: [2026, 2027]

  # Seasons to include. Options: winter, spring, summer, fall, or "all".
  # Default: all
  seasons: ["all"]

  # Maximum results per season from AniList. Default: 100.
  max_per_season: 100

  # Include ONA format alongside TV. Default: true.
  include_ona: true

  # Merge previous year's December shows into WINTER season.
  # When enabled, also fetches the prior year's WINTER and merges
  # any shows not already in the current year's results. This
  # captures December-premiering anime that AniList tagged under
  # the previous calendar year. Default: false.
  winter_overflow: false

  # AniList relation types to follow when a show is not found in
  # MDBList by its direct MAL ID. Default: [PREQUEL, PARENT]
  fallback_relation_types: [PREQUEL, PARENT]
    # PREQUEL   â€” previous season of the same show
    # PARENT    â€” broader franchise entry
    # ADAPTATION â€” source material (usually manga â€” use with caution)
    # SIDE_STORY â€” side stories (usually unrelated)

  # Exclude shows by AniList content tag name (case-insensitive).
  # Shows matching any tag in this list are skipped.
  exclude_tags: []

# MDBList list settings
mdblist:
  # List title template. Supports {season} and {year} placeholders.
  # Default: "Anime {season} {year}"
  title_template: "Anime {season} {year}"

  # List description template. Supports {season} and {year}.
  description_template: >
    All anime TV series airing in {season} {year},
    sourced from AniList. Generated by anilistgen.

  # Set to false to create private lists. Default: true.
  public: true

  # Shows to skip. Supports exact MAL IDs (numeric) or
  # case-insensitive title substrings.
  blacklist: []

# Logging
logging:
  # One of: debug, info, warn, error. Default: info.
  level: info
  # Path to log file. Empty means stderr.
  file: ""
```

## CLI modes

| Invocation | Mode | Behavior |
|---|---|---|
| `anilistgen` | **oneshot** | Runs once, processes all configured seasons, exits. |
| `anilistgen daemon` | **daemon** | Runs in background, repeats at `interval`, logs to file. |
| `anilistgen init-config` | **init** | Writes a default config file to the first config path or to `-config`. |
| `anilistgen validate` | **validate** | Validates config and tests API connectivity to AniList and MDBList. |

### Global flags

- `-config`, `-c PATH` â€” config file path (overrides default search).
- `-dry-run` â€” runs without making MDBList API calls. Prints what would
  be done.
- `-output`, `-o DIR` â€” writes JSON files per season instead of MDBList
  calls (oneshot only).
- `-v`, `-verbose` â€” more detailed logging.
- `-h`, `-help` â€” prints usage and exits.

### Oneshot mode

The default mode. Processes all configured seasons and years, then exits.
Prints list URLs to stdout on success.

### Daemon mode

`anilistgen daemon` enters a loop:
- Runs a full sync cycle (all configured seasons and years).
- Sleeps for `interval` duration.
- Repeats until SIGINT or SIGTERM (graceful shutdown with code 0).

If `interval` is 0 or omitted, daemon mode rejects startup with code 1.
If `run_on_start` is false, sleeps for the full interval before the first
sync cycle.

The tool writes a timestamp to `state_file` after each successful cycle.
A minimum 1-hour guard prevents rapid restarts from hammering APIs.

### Init mode

`anilistgen init-config` writes a documented default config to the first
config path in the search order. If `-config` is given, writes there.
Does not overwrite existing files unless `-force` is passed.

If no config exists and no `-config` is given, creates
`~/.config/anilistgen/anilistgen.yaml` and the parent directory.

### Validate mode

`anilistgen validate` loads the config, validates all fields, tests
connectivity to both AniList and MDBList APIs, then prints a summary.
Exits 0 on success, 1 on failure.

## Data source: AniList

### Query parameters

For each configured season and year, the tool queries the AniList GraphQL
API (`https://graphql.anilist.co`) for media matching:
- `type: ANIME`
- `season: {WINTER|SPRING|SUMMER|FALL}`
- `seasonYear: {year}`
- `format_in: [TV]` (and `ONA` if `include_ona` is true)

Results are sorted by AniList popularity (descending). The tool fetches
up to `max_per_season` results in a single page. If more results exist,
a warning is logged but no further pages are fetched.

### Fields returned per show

- `id` â€” AniList ID
- `idMal` â€” MyAnimeList ID (may be null)
- `title.english` / `title.romaji` â€” display title (English preferred)
- `format` â€” TV, ONA (TV_SHORT is excluded by the API query)
- `episodes` â€” episode count (may be null for airing shows)
- `duration` â€” minutes per episode (may be null)
- `genres` â€” genre strings
- `tags` â€” AniList content tags with name
- `status` â€” RELEASING, FINISHED, NOT_YET_RELEASED, etc.
- `relations` â€” related media edges (for fallback matching)

### Short show filtering

Shows are removed from results when:
- `duration` is â‰¤ 10 minutes per episode, OR
- `duration` and `episodes` are both known and their product (total
  runtime) is â‰¤ 10 minutes (single-episode shorts)

This catches both short-per-episode series (9-min episodes) and
single-episode shorts (6-min Ă— 1) while allowing series with short
episodes but many of them (9-min Ă— 12 episodes = legitimate series).

### Winter overflow

Because AniList labels Winter as Decemberâ€“February but assigns the
calendar year as `seasonYear`, a show premiering December 2025 is
tagged `WINTER 2025`, while a show premiering January 2026 is tagged
`WINTER 2026`. With `winter_overflow: true`, the tool fetches the
previous year's WINTER season as well and **merges** any shows not
already in the current year's results, deduplicated by AniList ID.

### Tag exclusion

When `exclude_tags` is set (e.g. `["Episodic", "Cute Girls Doing Cute Things"]`),
shows whose AniList tags match (case-insensitive) are filtered out before
publishing. Tags are fetched from AniList alongside genres.

### Fallback matching

When a show's direct MAL ID doesn't resolve in MDBList's database,
the tool traverses the show's AniList `relations` edges (e.g. PREQUEL,
PARENT) to find a related entry that MDBList does have. The show is
then added to the list under that related entry's provider ID.

The configured `fallback_relation_types` controls which relation types
are followed. Default: `[PREQUEL, PARENT]`. Adding `ADAPTATION` or
`SIDE_STORY` broadens matching but may include unrelated entries.

### Retry behavior

AniList API non-200 responses are retried up to 3 times with exponential
backoff (1s, 2s, 4s). If all retries fail, the season is skipped
gracefully â€” other seasons are still processed.

## MDBList output

### Item resolution pipeline

For each show that passes filtering:

1. **Batch MAL lookup** â€” All MAL IDs (direct + relation fallback) are
   collected and resolved via MDBList's `POST /mal/show` in a single
   batch call per season.
2. **Direct match** â€” If the show's own MAL ID is found, the returned
   IMDB ID (or TMDB/TVDB as fallback) is used as the provider ID.
3. **Fallback match** â€” If the direct MAL ID isn't found, the tool tries
   each configured relation type's MAL ID from AniList relations. The
   first match wins.
4. **Skip** â€” If neither direct nor fallback match found, the show is
   skipped with a debug log entry.

### List creation and update

Before creating a new list, the tool searches the authenticated account's
existing lists by title (case-insensitive). If a match is found:

- **Cache hit** (item cache has previous provider IDs for the list):
  Compute a diff between the cached IDs and the desired IDs.
  Remove stale items via `POST /lists/{id}/items/remove`, add new items
  via `POST /lists/{id}/items/add`. The list ID and URL are preserved.
- **Cache miss** (first sync or cache deleted): Delete and recreate the
  entire list, then populate the cache for future runs.

If no matching list exists, a new list is created via
`POST /lists/user/add` and items are added.

The item cache is stored at a JSON file alongside the state file
(e.g. `/tmp/anilistgen_listcache.json`) and is updated after each sync.

### Output modes

| Mode | Trigger | Behavior |
|---|---|---|
| Normal | `anilistgen` | Fetches AniList, resolves to MDBList, creates/updates lists. |
| Dry-run | `-dry-run` | Fetches AniList only, prints what it would do without API calls. |
| File output | `-output DIR` | Writes one JSON file per season with full show data (no MDBList calls). |

In all modes, after each MDBList API call, the tool waits at least 1.1
seconds. On HTTP 429 (rate limit), retries up to 3 times with exponential
backoff (2s, 4s, 8s).

## Season timing

AniList uses the northern-hemisphere meteorological seasons:

| Season | Months | Peak premiere window |
|---|---|---|
| WINTER | December â€“ February | Early January |
| SPRING | March â€“ May | Early April |
| SUMMER | June â€“ August | Early July |
| FALL | September â€“ November | Early October |

**Important**: The `seasonYear` on AniList is the **calendar year**.
`WINTER 2026` covers Januaryâ€“February 2026 shows. A December 2025
show is `WINTER 2025`, not `WINTER 2026`. Enable `winter_overflow: true`
to merge December shows from the prior year.

## Blacklist

The `blacklist` field supports:
- **MAL IDs**: numeric entries like `57658` skip by exact MAL ID.
- **Title substrings**: text entries match case-insensitively against
  the show's display title.
- **Tag exclusion**: use `exclude_tags` for AniList content tag matching.

## Exit codes

| Code | Meaning |
|---|---|
| 0 | Success (all configured seasons processed, or graceful shutdown) |
| 1 | Errors (invalid config, API failures â€” partial results may exist) |
| 2 | Invalid CLI flags |

## Invariants

- The tool never modifies MDBList lists whose title doesn't match the
  configured template.
- Re-running is always safe â€” the diff-based update is idempotent.
- Network failures are logged per-season; other seasons continue.
- Config file changes take effect on the next daemon cycle (no hot-reload).
- The item cache is the source of truth for diff updates. If deleted,
  the next sync performs a full replace.

## Deployment

### Binary

```bash
go build -o anilistgen ./cmd/anilistgen
./anilistgen init-config
vim anilistgen.yaml
./anilistgen -dry-run
./anilistgen
```

### systemd timer (one-shot)

```bash
sudo cp anilistgen /usr/local/bin/
sudo mkdir -p /etc/anilistgen
sudo cp anilistgen.yaml /etc/anilistgen/
sudo cp deploy/anilistgen.service /etc/systemd/system/
sudo cp deploy/anilistgen.timer /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now anilistgen.timer
```

### Docker (ghcr.io)

**Distroless, rootless image** â€” no shell, no package manager, runs as
non-root user. Multi-arch (linux/amd64 + linux/arm64).

`ghcr.io/calmcacil/anilistgen:latest`

#### Daemon (all config via env vars)

```bash
docker run -d --name anilistgen --restart unless-stopped \
  -e ALG_MDBLIST_API_KEY=your_key \
  ghcr.io/calmcacil/anilistgen:latest
```

#### One-shot

```bash
docker run --rm \
  -e ALG_MDBLIST_API_KEY=your_key \
  -e ALG_INTERVAL=0 \
  ghcr.io/calmcacil/anilistgen:latest anilistgen
```

#### With YAML config file bind mount

```bash
docker run -d --name anilistgen --restart unless-stopped \
  -v ./anilistgen.yaml:/etc/anilistgen/anilistgen.yaml:ro \
  ghcr.io/calmcacil/anilistgen:latest \
  -config /etc/anilistgen/anilistgen.yaml
```
